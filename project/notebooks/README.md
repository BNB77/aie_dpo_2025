# Ноутбуки проекта

В эту папку помещаются **экспериментальные ноутбуки**:

- разведочный анализ данных (EDA);
- прототипы моделей;
- отдельные эксперименты;
- разовые проверки гипотез.

Рекомендуется:

- давать ноутбукам осмысленные имена, например:
  - `exp01_eda_baseline.ipynb`
  - `exp02_model_tuning.ipynb`
- по возможности добавлять краткое текстовое описание в начале ноутбука:
  - цель эксперимента;
  - какие данные используются;
  - какие основные выводы.

Всё, что становится «боевым» кодом (препроцессинг, обучение, инференс), по возможности выносите в модульный код в папку `src/`.

---

## Ноутбуки этого проекта

- `01_eda.ipynb` — разведочный анализ открытого датасета UCI "Default of Credit Card Clients" (`data/credit_data_full.csv`): аномалии в сырых категориальных кодах, распределение целевой переменной, корреляции построенных признаков.
- `02_baselines.ipynb` — baseline (LogReg) vs тюнингованные RandomForest/GradientBoosting (GridSearchCV), калибровка вероятностей, подбор решающего порога, feature importance, обоснование выбора финальной модели.
- `figures/` — сохранённые графики (распределение классов, аномалии сырых данных, ROC-кривая, матрица ошибок, feature importance), используются в ноутбуках и в `report.md`.

### Зависимости для повторного запуска ноутбуков

`requirements.txt` покрывает только сервис и обучение моделей. Чтобы выполнить ноутбуки заново (а не просто открыть с готовыми выводами), дополнительно нужны:

```bash
pip install jupyter nbclient ipykernel
```
